# 🎨 وصف التصاميم التفصيلية للشاشات

هذا الوصف يشرح شكل كل شاشة بدقة لضمان تنفيذ احترافي.

## 🎨 المبادئ التصميمية العامة

### 1. نظام المسافات (Spacing Scale)
```
4px, 8px, 12px, 16px, 20px, 24px, 32px, 40px, 48px, 64px
```

### 2. نظام الأحجام (Typography Scale)
```
Display:    32px / bold    — للعناوين الكبيرة
Heading 1:  24px / bold    — للأقسام الرئيسية
Heading 2:  20px / semibold — للأقسام الفرعية
Body Large: 18px / regular — للنصوص البارزة
Body:       16px / regular — النص الأساسي
Small:      14px / regular — للنصوص الثانوية
Tiny:       12px / medium  — للتسميات والعدادات
```

### 3. نظام الظلال
```css
shadow-sm: 0 1px 2px rgba(0,0,0,0.05)
shadow:    0 4px 6px rgba(0,0,0,0.07)
shadow-md: 0 10px 15px rgba(0,0,0,0.1)
shadow-lg: 0 20px 25px rgba(0,0,0,0.1)
```

### 4. نظام الزوايا
```
small: 8px   — للأزرار الصغيرة
base:  12px  — للأزرار والمدخلات
lg:    16px  — للبطاقات
xl:    24px  — للنوافذ المنبثقة
```

## 🎭 حركات وتفاعلات (Animations)

### عند فتح الصفحة
- عناصر تظهر من الأسفل (slide up) بتأخير متتالي (stagger 50ms)

### عند الضغط على زر
- تأثير Scale 95% ثم العودة (spring)
- تأثير Ripple خفيف للأزرار الرئيسية

### عند تبديل التبويبات
- Fade out + Fade in خلال 200ms
- Underline للتبويب النشط يتحرك بـ Spring

### عند حفظ البيانات
- Toast يظهر من الأعلى
- لون أخضر + أيقونة ✓
- يختفي بعد 3 ثواني

### عند التحميل (Loading)
- Spinner بلون primary
- نص: "جاري المعالجة..."
- للـ AI: نص متغير: "جاري قراءة الكتاب..." ← "توليد الأسئلة..." ← "المراجعة النهائية..."

## 📱 الاستجابة (Responsive Design)

### نقاط الكسر (Breakpoints)
```
Mobile:       < 640px
Tablet:       640px - 1024px
Desktop:      > 1024px
```

### التعديلات حسب الحجم

**على الجوال:**
- الشريط الجانبي يصبح قائمة Hamburger
- بطاقات الفصول بعمود واحد
- الجداول قابلة للسحب الأفقي
- أزرار أكبر (min 44px height)

**على التابلت:**
- بطاقات الفصول بعمودين
- استخدام كامل للشاشة

**على الكمبيوتر:**
- بطاقات الفصول بـ 3-4 أعمدة
- شريط جانبي ثابت
- الجداول كاملة
