# 🤖 Prompt جاهز لـ Claude Code

## 📌 كيفية الاستخدام

1. افتح Claude Code في Terminal
2. أرفق الملفات الثلاثة:
   - `01_PRD_وثيقة_المواصفات.md`
   - `02_خرائط_التدفق.md`
   - `03_وصف_التصاميم.md`
3. انسخ الـ Prompt أدناه والصقه
4. اترك Claude Code يبني التطبيق

## 🎯 Prompt الأساسي (انسخه كما هو)

```
السلام عليكم Claude،

أبي منك تبني لي نموذج أولي (HTML Prototype) لتطبيق معلمين كامل 
وظيفياً. قبل البدء، اقرأ بعناية الملفات الثلاثة المرفقة:

1. 01_PRD_وثيقة_المواصفات.md — المواصفات الكاملة
2. 02_خرائط_التدفق.md — رحلة المستخدم
3. 03_وصف_التصاميم.md — التصميم البصري

## 🎯 المتطلبات التقنية

- **التقنيات:** HTML5 + CSS3 + JavaScript vanilla (بدون frameworks)
- **التخزين:** IndexedDB للبيانات الكبيرة، localStorage للإعدادات
- **الخطوط:** استخدم Tajawal من Google Fonts
- **الاتجاه:** RTL بالكامل
- **المكتبات المسموحة:**
  - jsPDF + html2canvas لتوليد PDF
  - SheetJS (xlsx) لقراءة ملفات Excel
  - Chart.js للرسوم البيانية
  - Anthropic SDK أو fetch API لـ Claude

## 🔑 التعامل مع الذكاء الاصطناعي

في ملف `js/ai-service.js`:
- استخدم مفتاح API شخصي يُحفظ في localStorage
- أضف شاشة إعدادات يضع فيها المعلم مفتاحه
- اعرض رسالة واضحة إذا المفتاح غير موجود
- استخدم Claude 3.5 Sonnet أو أحدث

## 📝 Prompts للذكاء الاصطناعي

اكتب في ai-service.js دوال لكل نوع من الطلبات:

1. generateExam(source, questionTypes, count, difficulty, notes)
2. generateWorksheet(topic, gradeLevel, notes)
3. generateEnrichment(lesson, style)
4. explainPage(pageContent, question)
5. solveQuestion(questionText, bookContext)
6. generateStrategyReport(name, date, class, description, imageCount)
7. generateInitiativeReport(name, date, beneficiaries, description)

استخدم System Prompts احترافية باللغة العربية تخصص Claude 
كمساعد تعليمي متخصص في المنهج السعودي.

## 🎨 مبادئ التصميم الصارمة

1. **اللغة:** كل النصوص بالعربية
2. **الاتجاه:** RTL في كل العناصر
3. **الألوان:** التزم بلوحة الألوان في ملف التصاميم
4. **الخطوط:** Tajawal فقط، مع احترام أحجام Typography Scale
5. **المسافات:** استخدم فقط القيم من Spacing Scale
6. **الحركات:** ناعمة (200-300ms)، ما تزعج
7. **الأيقونات:** استخدم أيقونات Unicode أو SVG بسيطة
8. **الظلال:** ناعمة، ما تكون ثقيلة
```

## 📌 Prompts إضافية (للمراحل اللاحقة)

### للتحسين بعد البناء الأول:

```
المشروع شغال بشكل جيد. الآن أبي تحسّنات:

1. راجع تجربة المستخدم في [اسم الشاشة] وحسّنها
2. أضف حركات (animations) أنعم للانتقالات
3. حسّن تصميم الطباعة لتكون احترافية أكثر
4. أضف شاشة "مساعدة" تشرح كيفية استخدام التطبيق
5. اختبر على شاشات صغيرة وحسّن responsive
```

### لإضافة ميزة جديدة:

```
أبي أضيف ميزة جديدة: [وصف الميزة]

الميزة تتكامل مع: [الشاشات المتأثرة]
المستخدم يستخدمها بهذا التدفق: [خطوات التدفق]
البيانات المطلوبة: [الحقول]

اقترح لي أفضل مكان لإضافة الميزة، ثم نفذها.
```

### لتحويل النموذج لتطبيق حقيقي (المرحلة التالية):

```
النموذج الأولي شغال تماماً. الآن أبي أحوله لتطبيق حقيقي.

الخيارات:
1. تطبيق ويب متقدم (PWA) مع React/Next.js
2. تطبيق جوال (React Native / Flutter)
3. تطبيق سطح مكتب (Electron)

أقترح لي الأفضل لحالتي، وابدأ في البناء بعد موافقتي.
```

## 💡 نصائح للعمل مع Claude Code

### 1. كن صبوراً في المرحلة الأولى
بناء 7 شاشات متكاملة يأخذ وقت. توقع 30-60 دقيقة للنموذج الأولي.

### 2. اختبر باستمرار
بعد كل مرحلة، اطلب من Claude Code أن يشرح لك ما فعله، واختبره بنفسك.

### 3. استخدم Git
احفظ تقدمك في Git بعد كل مرحلة ناجحة. لو حصل خطأ، ترجع بسهولة.

### 4. لا تخف من الطلب
إذا ميزة ما تشتغل، قل له صراحة: "الميزة X ما تشتغل، الخطأ هو كذا..."

### 5. ركز على الأهم
إذا تأخر المشروع، قل له: "ركز على الميزات الأساسية فقط، باقي الميزات لاحقاً"
